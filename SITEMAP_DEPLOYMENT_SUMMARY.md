## 🚀 What You Need to Do (5 Steps)

### **Step 1: Update Domain** (1 minute)
If deploying to a custom domain, update `.env`:

```bash
# Change this line:
VITE_SITE_URL=https://studenttoolhub.vercel.app

# To your domain:
VITE_SITE_URL=https://youractualsite.com
```

Or set via environment variable in your deployment platform:
- **Vercel:** Settings → Environment Variables
- **Netlify:** Site Settings → Build & Deploy → Environment
- **Self-hosted:** Export before build: `export VITE_SITE_URL=https://yourdomain.com`

### **Step 2: Build & Deploy** (2-5 minutes)
```bash
npm run build
# Sitemap auto-generates in postbuild

# Deploy to your platform
npm run deploy
# or: git push (if using GitHub Actions)
# or: Deploy via Vercel/Netlify dashboard
```

### **Step 3: Verify Sitemap** (1 minute)
```bash
# Check if accessible
curl https://yourdomain.com/sitemap.xml

# Should return HTTP 200 with valid XML
# Should show your domain, not studenttoolhub.com
```

### **Step 4: Submit to Google** (2 minutes)
1. Go to: https://search.google.com/search-console/
2. Select your domain property
3. Left sidebar → **Sitemaps**
4. Click → **Add a new sitemap**
5. Enter: `yoursite.com/sitemap.xml`
6. Click → **Submit**

### **Step 5: Submit to Bing** (2 minutes)
1. Go to: https://www.bing.com/webmasters
2. Select your site
3. Left dashboard → **Sitemaps**
4. Click → **Submit**
5. Enter: `yoursite.com/sitemap.xml`

## 🎓 How This Works (Technical Details)

### **The Process**

```
1. npm run build
   ↓
2. Vite builds React app → dist/
   ↓
3. postbuild hook triggers
   ↓
4. scripts/generate-sitemap.js runs
   ↓
5. Reads src/data/tools.js
   ↓
6. Gets domain from VITE_SITE_URL env var
   ↓
7. Generates public/sitemap.xml
   ↓
8. Writes 99 URLs with timestamps
   ↓
9. public/ files copied to dist/
   ↓
10. Deploy dist/ to hosting
    ↓
11. Sitemap live at https://yourdomain.com/sitemap.xml
    ↓
12. Google/Bing discover via robots.txt
```

### **Why It's Better Than Manual**

| Manual Sitemap | Your Auto-Sitemap |
|---|---|
| 📝 Manually edit XML | 🤖 Auto-generated from data |
| 😰 Easy to make errors | ✅ Zero errors (script validated) |
| 👨‍💻 Need to remember | 🔄 Runs every build |
| 🐢 Outdated quickly | 📅 Always current |
| 📦 Miss new tools | 🎯 Includes all tools |

## 🚨 Common Issues & Solutions

### **Issue: Sitemap shows wrong domain**
```bash
# Solution: Update VITE_SITE_URL
export VITE_SITE_URL=https://yourdomain.com
npm run build

# Verify:
grep "<loc>" public/sitemap.xml | head -1
```

### **Issue: Sitemap returns 404**
```bash
# Check if file exists
ls -lh dist/sitemap.xml
ls -lh public/sitemap.xml

# Redeploy if it's missing
npm run build && npm run deploy
```

### **Issue: Google doesn't find sitemap**
1. ✅ Wait 48 hours (Google's crawl delay)
2. ✅ Check robots.txt includes sitemap reference
3. ✅ Manually submit in Search Console
4. ✅ Use "Request Indexing" on a tool URL

### **Issue: Tools not in sitemap**
```bash
# Check if tool is in tools.js
grep "slug.*your-tool" src/data/tools.js

# Regenerate
node scripts/generate-sitemap.js

# Verify count increased
grep -c "<url>" public/sitemap.xml
```
